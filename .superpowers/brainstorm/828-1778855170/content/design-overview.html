数据流

1. 搜索流程
用户输入关键词 → useEverything 更新 search 状态 → debounce 300ms → GET /api/everything?search=... → 服务端转发 Everything HTTP → JSON 解析 → 更新 fileList 状态 → FileList 重渲染
2. 目录浏览流程
点击文件夹 → setCurrentPath → GET /api/everything?path=C:\folder → 服务端拼接 parent:"C:\folder" 查询 → 返回子文件和子文件夹 → 更新列表 + 面包屑
3. 下载流程
点击下载 → GET /api/everything/download?filepath=... → 服务端 stream Everything HTTP 文件 → 浏览器触发下载保存
4. 收藏夹
纯客户端 localStorage，存储路径数组。点击收藏路径 → setCurrentPath → 触发目录浏览

技术要点

设计决策

  • Next.js API 路由代理 Everything HTTP，解决 CORS
  • 环境变量 EVERYTHING_HTTP_URL 配置地址
  • useEverything hook 封装所有数据请求和状态
  • 目录树节点按需加载（展开时才请求子目录）
  • FILETIME 日期在服务端转换为 ISO 8601

注意事项

  • 大目录可能返回大量结果，需要分页
  • Everything HTTP 不支持直接"列目录"——用 parent: 查询模拟
  • 路径中包含特殊字符时需 URL 编码
  • 无文件预览（按你的选择跳过）
